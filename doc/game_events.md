# Game Events Documentation

This document created by AI describes the game events generated by `CrpgSaveStatisticsBehavior` and used by the quests feature.

## Overview

The `CrpgSaveStatisticsBehavior` tracks player combat actions during Bannerlord multiplayer matches and sends them to the backend as `CrpgGameEvent` objects. These events are batched and sent every 5 seconds (configurable via `FlushIntervalMilliseconds`).

## Event Structure

All events share the following base structure:

```csharp
public class CrpgGameEvent
{
    public int? UserId { get; set; }          // User ID of the player who performed the action
    public EventType Type { get; set; }       // Type of event (Hit, Kill, Block)
    public Dictionary<EventField, string>? EventData { get; set; } // Event-specific data
}
```

## Event Types

### 1. Hit Event (`EventType.Hit`)
Triggered when a player successfully hits another player or mount.

**User ID**: The attacker's user ID
**When**: On successful hit (not blocked)
**Not sent for**: Self-hits, team hits, blocked attacks

**Fields:**
- **Damage** (Required): Inflicted damage as numeric string
- **HitType** (Required): "Ranged" or "Melee"
- **TargetType** (Required): "Character" or "Mount"
- **DamageType** (Required): Damage type as string: "Blunt", "Cut", "Pierce", or "Invalid"
- **BodyPart** (Optional): Body part hit (see Body Part Values below)
- **WeaponClass** (Optional): Weapon class (see Weapon Class Values below)
- **ItemId** (Optional): cRPG item ID of the weapon used *(Not currently implemented in event generation)*

### 2. Kill Event (`EventType.Kill`)
Triggered when a player kills another player or mount.

**User ID**: The killer's user ID
**When**: On agent death (agent state = Killed)
**Not sent for**: Self-kills, team kills, non-player agents

**Fields:**
- **TargetType** (Required): "Character" or "Mount"
- **DamageType** (Required): Damage type of killing blow: "Blunt", "Cut", "Pierce", or "Invalid"
- **WeaponClass** (Optional): Weapon class of killing weapon
- **ItemId** (Optional): cRPG item ID of the killing weapon *(Not currently implemented in event generation)*
- **HitType** (Optional): "Ranged" or "Melee" (determined from weapon)
- **BodyPart** (Optional): Body part where killing blow landed

### 3. Block Event (`EventType.Block`)
Triggered when a player blocks an attack (with shield or weapon).

**User ID**: The blocker's user ID (not the attacker's)
**When**: On successful block (shield or weapon)
**Block detection includes**:
- Shield blocks (`AttackBlockedWithShield`)
- Weapon blocks (`MissileBlockedWithWeapon`)
- Parried attacks (`CombatCollisionResult.Parried`)
- Chamber blocks (`CombatCollisionResult.ChamberBlocked`)
- Regular blocks (`CombatCollisionResult.Blocked`)

**Fields:**
- **Damage** (Conditional): Damage blocked (inflicted damage) as numeric string. Only present for shield blocks.
- **HitType** (Required): "Ranged" or "Melee"
- **TargetType** (Required): "Character" or "Mount"
- **BodyPart** (Optional): Body part that would have been hit
- **WeaponClass** (Optional): Weapon class of the blocking item
- **ItemId** (Optional): cRPG item ID of the blocking item *(Not currently implemented in event generation)*

**Note**: For shield blocks, the system uses the blocker's offhand wielded item index to find the shield. For weapon blocks, it uses the currently wielded weapon (primary or offhand). Damage field is only recorded for shield blocks because weapon blocks do not have a damage value in the collision data.

## Field Values Reference

### HitType Values
- **"Ranged"**: Attack from a ranged weapon (bow, crossbow, throwing weapons)
- **"Melee"**: Attack from a melee weapon or unarmed

### TargetType Values
- **"Character"**: Human player character
- **"Mount"**: Mount (horse) belonging to a player

### DamageType Values

## Event Generation Rules

### General Rules
1. **Warmup Period**: No events are generated during warmup (`CrpgWarmupComponent.IsInWarmup == true`)
2. **Player Validation**: Both affector (attacker/blocker) and affected (target) must be valid players or player mounts
3. **Team Restrictions**: Self-hits and team hits/kills are filtered out (`affectedAgent.Team == affectorAgent.Team`)
4. **Block Priority**: Blocked attacks generate Block events instead of Hit events

### Hit Event Generation
```csharp
// Simplified logic
if (!isBlocked && validPlayers && !selfHit && !teamHit) {
    CreateHitEvent();
}
```

### Kill Event Generation
```csharp
// Simplified logic
if (agentState == AgentState.Killed && validPlayers && !selfKill && !teamKill) {
    CreateKillEvent();
}
```

### Block Event Generation
```csharp
// Simplified logic
bool isBlocked = attackCollisionData.AttackBlockedWithShield
    || attackCollisionData.MissileBlockedWithWeapon
    || attackCollisionData.CollisionResult == CombatCollisionResult.Blocked
    || attackCollisionData.CollisionResult == CombatCollisionResult.Parried
    || attackCollisionData.CollisionResult == CombatCollisionResult.ChamberBlocked;

if (isBlocked && validPlayers && !selfBlock) {
    CreateBlockEvent();
}
```

## Data Flow

1. **In-game Action**: Player hits/kills/blocks in Bannerlord
2. **Mission Behavior**: `CrpgSaveStatisticsBehavior.OnAgentHit()` or `OnAgentRemoved()` triggered
3. **Event Creation**: Event object created with relevant data
4. **Buffering**: Event added to in-memory buffer
5. **Batch Processing**: Every 5 seconds, buffer is flushed
6. **API Call**: Events sent to backend via `_crpgClient.CreateCrpgGameEventsAsync()`
7. **Storage**: Events stored in database for quest processing

## Quest Integration

Game events are used by the quests feature to track player achievements and progress. Example quest conditions:

- "Deal 1000 damage with swords" → Filter `Hit` events by `WeaponClass` containing "Sword", sum `Damage`
- "Get 10 headshot kills" → Filter `Kill` events by `BodyPart == "Head"`
- "Block 50 attacks with shield" → Filter `Block` events by `WeaponClass` containing "Shield"
- "Kill 20 players with ranged weapons" → Filter `Kill` events by `HitType == "Ranged"`
- "Deal 500 Pierce damage" → Filter `Hit` events by `DamageType == "Pierce"`, sum `Damage`

## Technical Notes

### Equipment Index Mapping
The behavior maps Bannerlord `EquipmentIndex` to cRPG `CrpgItemSlot` for item ID lookup:

| EquipmentIndex | CrpgItemSlot |
|----------------|--------------|
| Head           | Head         |
| Cape           | Shoulder     |
| Body           | Body         |
| Gloves         | Hand         |
| Leg            | Leg          |
| HorseHarness   | MountHarness |
| Horse          | Mount        |
| Weapon0        | Weapon0      |
| Weapon1        | Weapon1      |
| Weapon2        | Weapon2      |
| Weapon3        | Weapon3      |
| ExtraWeaponSlot| WeaponExtra  |

### Performance Considerations
- Events are buffered in memory and sent in batches (default 5 seconds)
- Fire-and-forget API calls (`_ = _crpgClient.CreateCrpgGameEventsAsync()`)
- Minimal computation during gameplay (most data comes directly from Bannerlord APIs)

### Limitations
- Item ID lookup depends on player's equipped items being correctly synchronized
- Weapon type for kills uses currently wielded weapon, not necessarily the killing weapon
- Body part detection depends on Bannerlord's collision data accuracy
- Damage field for block events is only recorded for shield blocks, not weapon blocks
- Team hit/kill filtering is enabled (team hits are not recorded)
